# Final Operating Guide

## 每天流程

1. 打開 `/start-here` 或直接看 `/today`。
2. 有銀行、信用卡、分期、支出、股票、創業想法，貼到 `/intake`。
3. 想討論策略，先到 `/advisor-chat`。
4. 需要跨分支總管判斷，到 `/command-brain`。
5. 所有草稿都回到 `/review-queue` 審核。

## 財務資料怎麼貼

銀行、現金、信用卡、分期、股票、支出可以用自然文字貼到 `/intake` 的整批資料輸入。系統只建立 draft，不自動投資、不自動付款。

## 支出怎麼貼

只貼大筆、警訊、信用卡、分期、創業測試或會影響安全現金水位的支出。信用卡繳費不 double count 為新消費。

## 股票怎麼問

到 `/advisor-chat` 選投資決策，或到 `/investment-decisions` 建立投資決策 draft。系統不會直接叫你買或賣，只列條件、風險、停損與缺資料。

## 客戶課表怎麼記

到 `/client-ops` 建立 client profile 或 session note draft。只記錄訓練重點、反應、注意事項與下一堂計畫，不做醫療診斷、不自動傳訊息。

## 內容怎麼產

到 `/content-studio` 建立 IG post、國考筆記、PDF 商品或文章草稿。所有內容都需要 Mark review，不自動發布。

## 創業怎麼評估

到 `/business-lab` 建立 business experiment 或 startup test plan，必須有 test budget、validation method、stop loss。系統不聯絡供應商、不下單、不付款。

## Codex 工作怎麼建立

到 `/product-roadmap` 建立 product feature draft，或到 `/codex-jobs` 查看 Codex job drafts。系統不自動 PR、不自動 deploy。

## 目前不做

- 不 deploy functions。
- 不啟用 LINE reply / push。
- 不自動交易、付款、下單、聯絡供應商、發布內容、傳訊息給客戶。
- 不做醫療診斷。
- 不假裝有即時股價。
