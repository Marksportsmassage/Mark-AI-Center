# Do Not Ask Again

Updated: 2026-06-27

The assistant should not repeatedly ask Mark for these items unless it needs confirmation that the data changed.

## Do Not Ask Again Unless Changed

- Whether the first finance baseline is usable: Mark confirmed it.
- Whether credit card and installments are duplicates: Mark clarified they are separated.
- Whether student loan exists: Mark confirmed approximate balance and deferment.
- Whether monthly living cost exists: Mark gave an initial 15,000-20,000 TWD estimate.
- Whether investment decisions are approved: they are not approved and remain waiting_review.
- Whether average down is allowed: it remains false unless Mark gives full conditions.
- Whether LINE reply / push can be enabled: no, not without explicit future approval.
- Whether functions deploy is allowed: no, not for current work.
- Whether raw PDFs should be committed: no.
- Whether raw finance/card/bank data should be posted to GitHub: no.

## Ask Only If Context Requires Update

- Has monthly living cost changed?
- Has credit card / installment status changed?
- Has student loan deferment status changed?
- Has Mark approved income offer prices?
- Has Mark approved any client-facing script?
- Has Mark approved LINE reply / push or functions deploy?
